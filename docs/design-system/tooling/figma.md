# Figma

Figma represents design intent and handoff context.

It should help teams align on anatomy, variants, states, layout, and token
intent before implementation. It should not be treated as the authoritative
production token source unless the team formally chooses and validates that
workflow.

## Figma Should Capture

- Component anatomy
- Supported variants
- Interaction and validation states
- Token naming alignment
- Accessibility and content notes
- Links to implementation and documentation where available

## Questions To Validate

- Are Figma variables the source of token changes or a design reference?
- How are token updates reviewed and promoted into code?
- Which component states must be represented before development begins?
- How are shipped implementation differences reported back to design?
