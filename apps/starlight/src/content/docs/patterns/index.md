title: Patterns
description: Composed component guidance for recurring public-service workflows and application structures.
---

Patterns explain how governed components work together to solve recurring user needs. They are not additional component variants and should not duplicate component documentation.

## Pattern principles

- begin with a user or service-delivery need;
- compose approved public component contracts;
- document content, validation, focus, responsive, and error behavior;
- distinguish required structure from adaptable examples;
- use realistic public-service language and data density;
- link back to each component contract and its known limitations.

Pattern pages will follow the flagship component work so their guidance is built on stable, documented primitives rather than sample-only UI.
