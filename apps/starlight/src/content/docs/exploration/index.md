Exploration records how the system was discovered and improved without forcing implementation history into every component page.

## What belongs here

- duplicate component contracts and competing API philosophies;
- selector and naming inconsistencies;
- provider leakage and wrapper escape hatches;
- missing canonical stories or incomplete test evidence;
- accessibility gaps and remediation outcomes;
- Figma-versus-code differences;
- rejected approaches and architectural tradeoffs;
- historical documentation-platform experiments.

The first detailed case study will follow the Button work and show how shipped behavior, candidate design intent, Storybook evidence, accessibility expectations, and the public API were reconciled.
